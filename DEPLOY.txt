Hello Elementor Child — deploy checklist
========================================

Requirements on the server
---------------------------
- WordPress + WooCommerce active
- Parent theme: Hello Elementor (folder name must be hello-elementor)
- ACF (Free or Pro) active — product / variation fields register from functions.php
- Variable products need a can-count attribute (default slug filter: pa_can-count)

Install
-------
1. Upload this entire folder to: wp-content/themes/hello-elementor-child/
2. WP Admin → Appearance → Themes → Activate "Hello Elementor Child"
3. Clear any host / plugin caches

After deploy
------------
- Edit products: "HtoEAU Product" and per-variation "HtoEAU Variation" meta boxes
- PDP: custom gallery, packs, subscribe vs one-time, accordions, sample kit + transformation sections

Optional — extra content below the PDP (sample kit + transformation)
--------------------------------------------------------------------
- WP Admin → Appearance → Widgets → find “Product page — below content”
  Add any block/widget; it renders after the transformation section on single product pages.
- Elementor Pro: Theme Builder can target location “HtoEAU — Below product content” (same slot;
  if an Elementor template is assigned there, it is used instead of the widget area).

Optional — conflicts
--------------------
- If Elementor Theme Builder has a full Single Product template, it can conflict with PHP overrides.
  Prefer one approach: either this child theme templates OR Elementor single-product template.
